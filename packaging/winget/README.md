# winget submission for raggity (portable)

raggity ships to winget as a **portable** package: the Windows release zip is
extracted and `rag.exe` is exposed via a command alias. The manifest itself is
generated by [`wingetcreate`](https://github.com/microsoft/winget-create) at
submit time, so there is no manifest checked in here — only the exact commands.

- **PackageIdentifier:** `IxMxAMAR.raggity`
- **InstallerType:** `zip` with `NestedInstallerType: portable`
- **PortableCommandAlias:** `rag`
- **Installer URL:** `https://github.com/IxMxAMAR/raggity/releases/download/v<VERSION>/raggity-<VERSION>-windows-x86_64.zip`

## First submission (new package)

Install wingetcreate (`winget install wingetcreate`), then:

```powershell
$ver = '0.10.0'
wingetcreate new "https://github.com/IxMxAMAR/raggity/releases/download/v$ver/raggity-$ver-windows-x86_64.zip" `
  --version $ver `
  --out .\out
```

wingetcreate downloads the zip, computes the SHA256, and walks you through the
metadata prompts. When prompted, set:

- Installer type: **zip**
- Nested installer type: **portable**
- Nested installer file — RelativeFilePath: **`rag.exe`**, PortableCommandAlias: **`rag`**
- PackageIdentifier: **`IxMxAMAR.raggity`**
- Publisher: `IxMxAMAR`, PackageName: `raggity`, License: `AGPL-3.0`
- Moniker: `raggity`, ShortDescription: "Local-first, top-tier RAG over your notes, docs and PDFs — answered by Claude."

Then submit the pull request to microsoft/winget-pkgs:

```powershell
wingetcreate submit --token <GITHUB_PAT> .\out
```

## Version bumps (existing package)

Once `IxMxAMAR.raggity` exists in winget-pkgs, updates are one command:

```powershell
$ver = '0.11.0'
wingetcreate update IxMxAMAR.raggity `
  --version $ver `
  --urls "https://github.com/IxMxAMAR/raggity/releases/download/v$ver/raggity-$ver-windows-x86_64.zip" `
  --submit --token <GITHUB_PAT>
```

`--submit` opens the PR automatically after regenerating the manifest and
checksum.
